# Blog + Full Site CMS Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Integrate Sanity CMS to manage all site content (hero, services, portfolio, team, pricing, footer social links) and add a blog with RO/EN/RU support.

**Architecture:** Next.js 16 App Router pages become async Server Components that fetch from Sanity CDN via GROQ. Client Components that need Sanity data receive it as props from their parent Server Components. Blog posts use `@sanity/document-internationalization` (separate documents per locale). All other schemas use inline `{ ro, en, ru }` locale fields.

**Tech Stack:** `next-sanity`, `@sanity/client`, `@sanity/document-internationalization`, `@portabletext/react`, `@sanity/image-url`, `sanity`

**IMPORTANT — Read before writing any code:** Check `node_modules/next/dist/docs/` for Next.js 16 APIs. `params` is `Promise<{...}>` and must be `await`-ed. Async Server Components must use `getTranslations`/`getLocale` from `next-intl/server`, NOT `useTranslations`/`useLocale` hooks. Tailwind v4 — no `tailwind.config.ts`, use `var(--color-*)` CSS variables, never `dark:` classes.
